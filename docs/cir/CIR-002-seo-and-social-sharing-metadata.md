# CIR-002: Add SEO and social-sharing metadata

## Intent

Make every page advertise itself well to search engines and to the link
previews generated by social platforms and messaging apps. Before this
change the `<head>` carried only a title and, on some pages, a
description — so a shared link rendered as bare text with no card, search
engines had no canonical anchor to the custom domain, and nothing tied the
site's pages to their author as an identifiable entity.

## Behavior

- GIVEN any page
- WHEN it is rendered
- THEN it emits a `description`, a `<link rel="canonical">` pointing at its
  own permalink, an `author` tag, and a full set of Open Graph and Twitter
  Card tags

- GIVEN a page whose front matter sets no `description`
- WHEN its meta tags are built
- THEN the description falls back to the site-wide `params.description`

- GIVEN the home page
- WHEN a search engine reads its structured data
- THEN it finds a schema.org `Person` whose `sameAs` links the site owner's
  GitHub and LinkedIn profiles

- GIVEN a blog post
- WHEN a search engine reads its structured data
- THEN it finds a schema.org `BlogPosting` with headline, description,
  publication date, and author

- GIVEN a page that is neither the home page nor a single post (the blog
  list, a tag term page)
- WHEN its structured data is built
- THEN it emits no JSON-LD, because there is nothing useful to assert, and
  its Open Graph type is `website` rather than `article`

- GIVEN a page or post that sets no image
- WHEN it is shared
- THEN it presents the site-wide `og-default.png` card

- GIVEN a post that sets its own `ogImage` in front matter
- WHEN it is shared
- THEN that image is used in place of the default

## Constraints

- Match the existing template idiom: Hugo partials included from
  `layouts/_default/baseof.html`, driven by values in `hugo.toml`, with no
  new build tooling or third-party theme.
- The default sharing card had to be generated from the existing local
  toolchain (`rsvg-convert`, already present for the resume pipeline) and
  use the site's Modus colour palette so it reads as part of the brand.
- Preserve the WCAG AAA work from CIR-001: the change adds only `<head>`
  metadata and a static image, touching no rendered, interactive, or
  visible content.

## Decisions

- **Two shared partials rather than per-template tags.** The metadata is
  identical in shape across the home page, list pages, and posts — only a
  few values vary by page kind (`og:type`, the presence of article times
  and JSON-LD). Centralising this in `head-meta.html` and
  `head-schema.html` keeps the variation in one place; scattering the tags
  across `single.html`, `list.html`, and `index.html` would have
  duplicated them and invited drift. The page-kind branches live inside the
  partials, keyed on `.IsHome` / `.IsPage`.

- **A single site-wide sharing card rather than per-post generated
  images.** Per-post cards (rendering the title into an image at build
  time) are higher effort and add a build dependency for a seven-post site.
  The partial already honours a per-post `ogImage` front-matter value, so
  individual posts can opt into a custom image later without reworking the
  templates. This is deliberately reversible.

- **`sameAs` profile links carried in site params.** Emitting the GitHub
  and LinkedIn URLs as schema.org `sameAs` is what lets a search engine
  associate the site with a known person. Storing them in
  `params.social` keeps them out of the template markup and reusable by
  both the `Person` and `BlogPosting` author node.

- **`article:modified_time` currently mirrors `published_time`.** Posts
  carry no `lastmod`, so Hugo reports the publication date as the
  last-modified date. Left as-is rather than suppressed: it is standard
  Hugo behaviour and self-corrects the moment a post gains a real
  `lastmod`.
